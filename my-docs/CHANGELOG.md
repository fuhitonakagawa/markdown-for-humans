# CHANGELOG（Local Fork）

### Added（追加）

- **複数エディタ対応の強化**
  - 同一Markdownドキュメントを複数エディタで開いた場合のWebview同期管理を改善。
  - エコー更新抑制や同期の安定性に関するテストを追加。

- **UpNote風ショートカット対応**
  - WYSIWYG編集でのショートカット入力を追加し、キーボード操作の効率を改善。

- **添付ファイルの取り扱い強化**
  - 画像だけでなく添付ファイル（例: PDF）を含むドラッグ＆ドロップ処理を拡張。
  - 添付保存とリンク生成の経路を整理。

- **ファイルリンク処理の改善**
  - ファイルリンク解決や添付連携の挙動を改善し、リンク運用の一貫性を向上。

- **エディタズーム機能**
  - `Zoom In / Zoom Out / Zoom Reset` を追加。
  - WYSIWYGの表示倍率を操作できるようにし、可読性調整を容易化。

- **Markdownテキストエディタへ戻る導線**
  - カスタムエディタから既定Markdownエディタへ戻すコマンドを追加。
  - `editor/title` からの切り替え導線を実装。

- **拡張設定オープン機能の改善**
  - 設定ボタンから拡張設定を開く処理を追加・改善。
  - 拡張IDの扱いを見直し、設定遷移の安定性を向上。

### Changed（変更）

- **画像/添付の既定保存先を `Files` に統一**
  - 初期設定時の保存フォルダを分かりやすい既定値へ変更。

- **画像保存先解決ロジックをリファクタリング**
  - `resolveAutoImageTargetFolder` を追加し、ドラッグ＆ドロップ時の保存先決定を整理。

- **ローカルFork向け表示名/コマンド表記を調整**
  - `package.json` の表示名やコマンドタイトルを local fork 前提で明確化。

- **エディタタイトルのアイコン調整**
  - `editor/title` のコマンドアイコンを調整し、導線の視認性を改善。

### Removed（削除）

- **ツールバー上の「reopen in default editor」ボタンを削除**
  - 機能重複を避けるため、ツールバー側の導線を整理（タイトルアクション側に統合）。
